# Project: Client-Side Text Case Converter

## Project Overview
This project is a fully client-side Text Case Converter web tool designed as a micro-SaaS product.

The tool allows users to paste text or upload files containing text, edit the content directly in the browser, and convert text cases easily (uppercase, lowercase, title case, sentence case, etc.).

The project must look professional, beginner-friendly, SEO-optimized, and suitable for selling on marketplaces like Flippa.

No backend, server, or API is allowed.

---

## Core Principles
- 100% client-side execution
- Beginner-friendly UX
- Clean, modern SaaS-style UI
- SEO-first structure
- Shared hosting compatible (Hostinger)
- Lightweight and fast

---

## Technology Stack
- HTML (semantic, SEO-friendly)
- CSS (modern, responsive, professional styling)
- Vanilla JavaScript (no frameworks)
- Optional: lightweight client-side libraries ONLY if strictly necessary

---

## Agent Responsibilities

### 1. UI/UX Agent
**Goal:** Create a premium SaaS-style interface.

Responsibilities:
- Design a clean card-based layout
- Use professional color palettes
- Ensure strong contrast and readability
- Mobile-first responsive design
- Clear buttons, icons, and spacing
- Beginner-friendly interface with no clutter

---

### 2. Frontend Logic Agent
**Goal:** Handle all text processing in the browser.

Responsibilities:
- Handle pasted text
- Handle drag & drop file uploads
- Extract text from supported files (client-side)
- Enable text selection inside editor
- Apply case changes to:
  - Entire text
  - Selected text
  - Specific words (including repeated instances)
  - Specific sentences
- Ensure fast, smooth interaction

---

### 3. SEO & Content Agent
**Goal:** Make the tool rankable on search engines.

Responsibilities:
- Proper semantic HTML structure
- SEO-optimized page title and meta description
- Clear H1, H2 hierarchy
- Short, simple explanations for beginners
- FAQ section with real user intent questions
- Fast-loading layout

---

### 4. Accessibility & Usability Agent
**Goal:** Make the tool usable for everyone.

Responsibilities:
- Keyboard-friendly controls
- Readable font sizes
- Clear focus states
- Helpful placeholders and tooltips
- Easy first-time user experience
